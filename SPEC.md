# LayoutSwitcher — спецификация

Версия: 1.1 · Платформы: macOS (Swift), Windows (Go)

## 1. Назначение

Лёгкая фоновая утилита, заменяющая три сторонние программы одной:

| Заменяет | Функция |
|----------|---------|
| Punto Switcher | Смена раскладки клавиатуры по горячей клавише |
| Lightshot | Скриншот выделенной области с сохранением и копией в буфер |
| AnyDesk (частично) | — (удалён за ненадобностью) |

Принцип: не свой движок, а тонкая обёртка над **встроенными инструментами ОС**. Поэтому вес килобайты, нет телеметрии, надёжность как у системных средств.

## 2. Горячие клавиши

| macOS | Windows | Действие |
|-------|---------|----------|
| `Cmd+Shift` | `Ctrl+Shift` | Переключить раскладку (следующая в списке) |
| `Cmd+\` | `Ctrl+\` | Скриншот области → файл + буфер + плашка |
| `Cmd+Shift+\` | `Ctrl+Shift+\` | Видео: нативная панель записи |
| `F12` | — | То же видео (если F-клавиши в обычном режиме) |

Клавиша скриншота на Mac меняется без пересборки:
`defaults write com.nikita.layoutswitcher screenshotKeycode 42` (42=`\`, 50=`` ` ``, 14=`е`)

## 3. Поведение по функциям

### Смена раскладки
Ловится момент, когда модификаторы (Cmd+Shift / Ctrl+Shift) зажали и **отпустили без других клавиш**. Если между нажатием и отпусканием была любая клавиша — раскладка не переключается (это была другая комбинация). Логика одинаковая на обеих ОС.

### Скриншот
1. Выделение области (нативный селектор ОС)
2. macOS: сохранение в `~/Screenshots/ss_<дата>.jpg`, сжатие в JPEG (качество 60%)
3. Копирование в буфер обмена (вставка через Cmd/Ctrl+V)
4. Звук подтверждения (Glass)
5. Плашка-уведомление внизу экрана с именем файла и размером

### Видео
Открывается нативная панель записи ОС (macOS: Cmd+Shift+5, Windows: Game Bar). Там пользователь выделяет область, включает звук при необходимости, останавливает запись через системный контрол.

## 4. Уведомление (HUD-плашка)

Своё окно вместо системных уведомлений (они глушатся у фоновых приложений).
- Тёмная непрозрачная плашка внизу по центру экрана
- Контрастный светлый текст, синяя рамка, бирюзовая акцентная полоска слева
- Появляется на 3.3 секунды с плавным fade
- Не перехватывает мышь (ignoresMouseEvents)

## 5. Архитектура

### macOS (основная, в проде)
- `src/main.swift` (~270 строк), Swift + Carbon/AppKit
- Перехват клавиш: `CGEvent.tapCreate` (event tap)
- Раскладка: `TISSelectInputSource`
- Скриншот: `/usr/sbin/screencapture`
- HUD: `NSWindow` + `NSView` со слоями
- Сборка → `.app` bundle → `/Applications/LayoutSwitcher.app`
- Автозапуск: LaunchAgent `com.nikita.layoutswitcher` (KeepAlive, поднятие при сбое)

### Windows (Go, общее ядро)
- `cross/main.go` — общая логика (одна на обе ОС)
- `cross/platform_windows.go` — Win32 через syscall
- `cross/platform_darwin.go` — Mac-заглушка ядра (build-теги)
- Перехват клавиш: low-level keyboard hook (`WH_KEYBOARD_LL`)
- Раскладка: `WM_INPUTLANGCHANGEREQUEST`
- Скриншот: эмуляция Win+Shift+S (Snipping)
- Видео: эмуляция Win+Alt+R (Game Bar)
- Сборка: кросс-компиляция в `.exe` (x64 + ARM64)

## 6. Права macOS

Два разрешения в Системные настройки → Конфиденциальность:
- **Универсальный доступ** (Accessibility) — перехват клавиш
- **Запись экрана** (Screen Recording) — скриншоты и видео

⚠️ Каждая пересборка меняет cdhash (ad-hoc подпись), macOS сбрасывает права — выдавать заново.

## 7. Дистрибуция

- Репозиторий: https://github.com/ZnaNik/LayoutSwitcher
- Windows-сборки: https://github.com/ZnaNik/LayoutSwitcher/releases/latest (x64 + ARM64)
- CI: GitHub Actions собирает Windows-бинарники при пуше тега `v*`

## 8. Известные ограничения

- Windows-версия собрана кросс-компиляцией, на живой Windows ещё не обкатана
- F12 на MacBook по умолчанию media-клавиша (нужен режим обычных F-клавиш)
- Скриншот/видео требуют Screen Recording — без него тихо не работают
- На Mac смена раскладки в Go-ядре не реализована (используется Swift-версия)

## 9. Тестирование Windows с Mac

Только виртуальная машина (Parallels / UTM) с Windows 11 даёт живую сессию для проверки горячих клавиш. На Apple Silicon — Windows 11 ARM, брать `arm64` сборку. Wine и CI для GUI-хоткеев не подходят.
